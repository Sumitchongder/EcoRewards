# Security Policy

## Overview

EcoRewards is a frontend-based web platform built with HTML, CSS,
and JavaScript. While it does not currently handle sensitive backend
operations, security and responsible behavior remain important.

---

## Supported Versions

Only the latest version of the project is actively maintained.
Security fixes, if required, will be applied to the most recent version.

---

## Reporting a Security Issue

If you discover a security vulnerability, please do NOT open a public issue.

Instead, report it responsibly by contacting the project maintainer
directly via email.

Please include:

- A clear description of the issue
- Steps to reproduce the vulnerability
- Potential impact (if known)
- Screenshots or code snippets (optional)

---

## In-Scope Issues

- Client-side security flaws
- JavaScript injection vulnerabilities
- Data exposure risks (if future features add data handling)
- UI-based phishing or spoofing risks

---

## Out-of-Scope Issues

- Issues caused by third-party browser extensions
- User device or browser misconfiguration
- Denial-of-service attacks outside frontend scope

---

## Responsible Disclosure

We request that security researchers:

- Act in good faith
- Avoid public disclosure before a fix is available
- Limit testing to what is necessary to demonstrate the issue

---

## Disclaimer

EcoRewards is currently a frontend demonstration platform.
It should not be used for real financial transactions without
additional backend security, authentication, and validation layers.
